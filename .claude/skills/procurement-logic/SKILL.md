# 발주 로직 스킬

## 발주 프로세스

### 1. 자동발주 생성 프로세스
```
1. 전 제품 현재고 vs 발주점 비교
2. 발주 필요 제품 필터링 (현재고 < 발주점)
3. 발주량 결정 (EOQ 또는 목표재고 - 현재고)
4. 우선순위 스코어링
5. 발주서 생성 (공급사별 그룹핑)
```

### 2. 발주 우선순위 스코어 (100점 만점)
| 요소 | 배점 | 기준 |
|------|------|------|
| 재고 긴급도 | 40점 | 현재고/안전재고 비율 역수 |
| ABC 등급 | 25점 | A:25, B:15, C:5 |
| 판매 추세 | 20점 | 최근 4주 vs 이전 4주 증감율 |
| 리드타임 | 15점 | 리드타임 길수록 높은 점수 |

### 3. EOQ (경제적 발주량)
```
EOQ = √(2 × 연간수요 × 주문비용 / 재고유지비용)
```

### 4. 정기발주 수량
```
발주량 = 목표재고 - 현재고 - 미입고발주량 + 예상판매량(리드타임 동안)
```

### 5. 발주 시뮬레이션
- 향후 N일간 재고 변동 시뮬레이션
- 발주 타이밍별 재고 수준 시각화
- 품절 리스크 계산
- 비용 최적화 시나리오 비교

## 규칙
- 발주량은 최소발주단위(MOQ)의 배수로 올림
- 발주금액 = 발주수량 × 단가 (세금 별도)
- 공급사별 최소발주금액 체크
- 리드타임은 영업일 기준
- 발주 상태: 대기 → 승인 → 발주 → 입고 → 완료
- 모든 금액은 원화(KRW) 표시
